# Deep Graph Anomaly Detection with Contrastive Learning

Unsupervised anomaly detection on graph-structured data using Graph Autoencoders (GAE)
and SimCLR-inspired contrastive learning on the 7 TUDataset benchmark datasets.

This project was developed as part of the anomaly detection framework proposed for
[ML4SCI GENIE — GSoC 2025](https://github.com/felipetp-ctrl/gsoc-ml4sci-genie-submission).

## About

A partial reimplementation inspired by the paper:

> **Deep graph level anomaly detection with contrastive learning**
> Luo, X. et al. (2022). *Scientific Reports*, 12, 19867.
> DOI: [10.1038/s41598-022-22086-3](https://doi.org/10.1038/s41598-022-22086-3)

The original GLADC framework proposes a two-stage unsupervised approach: a Graph Autoencoder
baseline and a SimCLR-based contrastive model with a hypersphere-inspired anomaly score.
This reimplementation adapts the core ideas with architectural differences described below,
and benchmarks them across several TUDataset graph classification datasets.

## Architecture

This reimplementation follows the GLADC protocol with the following design choices:

### Graph Autoencoder (GAE) — baseline
- Encoder uses **EdgeConv** layers instead of the plain GCN used in the original paper
- Decoder reconstructs the adjacency matrix via sigmoid(Z @ Z.T) per graph
- Loss: **MSE reconstruction error** (instead of BCE, which proved less stable)
- Anomaly score: negative reconstruction error (lower error = more normal)

### SimCLR on Graphs — main model
- Augmentation strategy: explicit **node feature dropout (p=0.1)** and **edge dropout (p=0.1)**,
  applied to generate two views per graph — in contrast to the weight perturbation used in the
  original paper
- Encoder: EdgeConv-based GNNEncoder returning both **node-level** and **graph-level** embeddings
- Projection head: MLP (2×hidden → proj_dim) with BatchNorm, NT-Xent loss at τ = 0.5
- Anomaly score: hybrid node + graph distance from training centroid, inspired by **Equation 11**
  of the original paper: `score = mean_node_error + graph_error`, where errors are squared
  L2 distances to the respective centroids

### Training protocol
- Stratified train/val/test split (70/15/15)
- For datasets without node features, **node degree** is used as the sole node feature (1-dim)
- Optimizer: **AdamW** with decoupled weight decay
- Early stopping with patience = 10 on validation loss

## Overview

The project implements two unsupervised anomaly detection approaches for graphs:

**Graph Autoencoder (GAE) — baseline**
- GNN encoder produces node-level embeddings
- Decoder reconstructs the adjacency matrix via inner product
- Anomaly score = MSE reconstruction error per graph

**SimCLR on Graphs — main model**
- Two augmented views of each graph are generated via node feature dropout and edge dropout
- A GNN encoder is trained with NT-Xent contrastive loss to produce similar embeddings
  for views of the same graph and dissimilar embeddings for different graphs
- Anomaly score = hybrid node + graph distance from the centroid of the training distribution

## Architecture Details

### GNN Encoder
- 2 EdgeConv layers (hidden dim = 64) with BatchNorm
- Global mean + max pooling concatenated → 128-dim graph embedding
- Returns both node-level and graph-level representations
- Same architecture used in the ML4SCI GSoC submission (Common Task 2)

### SimCLR Model
- GNN Encoder + Projection Head (128-dim → 128-dim MLP with BatchNorm)
- NT-Xent loss with temperature τ = 0.5
- Graph augmentations: node feature dropout (p=0.1) + edge dropout (p=0.1)

### GAE
- GNN Encoder (node-level, no pooling)
- Decoder: sigmoid(Z @ Z.T) per graph
- MSE loss on adjacency reconstruction
